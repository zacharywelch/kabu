Email validation w/ Mailgun

Inspired by Luke Wroblewski's How to Reduce E-Mail Input Errors, this tutorial looks at advanced email validation using mailgun and Kabu. Along the way we'll add several enhancements to improve the form process and validation feedback.

1. Create a navbar

Start by creating a navbar. Let's add links to Sign in and Sign up and give it a background-fill of .fill-denim. In our example we'll only focus on Sign up.

4. Creating feedback

One of the great features of Mailgun is that it returns suggestions in case the email was mistyped. We'll be integrating with Mailgun shortly but before that let's design what feedback should look like when an email has suggestions.

To associate feedback with the input it belongs to we'll position the suggestion below Username and darken its background with a subtle .fill-darken. Later we'll add javascript so clicking the suggestion updates the Username.

5. Integrating with Mailgun

Mailgun is a popular email service for developers and used at large companies such as GitHub, Lyft, Stripe and Heroku. We'll use their open-source jQuery plugin to make our requests and provide feedback for users. See the Mailgun website to learn more about this powerful service.

Each request to mailgun returns a status of whether the email is valid. If the email is valid, mailgun may also return suggestions in case the user mistyped their email. We'll use our feedback markup from Step 4 and add it dynamically when there's a suggestion. Try typing in example@gnail.com and notice how clicking the suggestion updates Username

6. One final touch

Our feedback is getting displayed but it's a little jarring to have it suddenly appear and readjust the form. Let's use motion to build meaning into the feedback and communicate its spatial relationship with Username. We'll use .slideDown and have feedback transition from Username, making it clear how the two are associated and the form adjustment smoother.
